# 如何在 Windows 上运行项目

## 第一步：打开 PowerShell 或命令提示符

### 方法 1：在项目文件夹直接打开（最简单）

1. **打开文件资源管理器**（Win + E）
2. **导航到项目文件夹**：`F:\webtest`
3. **在地址栏输入并回车**：
   ```
   powershell
   ```
   或者
   ```
   cmd
   ```

4. **或者**：在文件夹空白处按住 **Shift** + **右键**，选择：
   - "在此处打开 PowerShell 窗口" 
   - 或 "在此处打开终端窗口"

### 方法 2：使用 Windows 搜索

1. **按 Win 键**（键盘左下角的 Windows 图标）
2. **输入** `powershell` 或 `cmd`
3. **点击打开** "Windows PowerShell" 或 "命令提示符"
4. **切换到项目目录**：
   ```
   cd F:\webtest
   ```
   注意：如果路径包含中文，可能需要使用引号：
   ```
   cd "F:\webtest"
   ```

### 方法 3：使用快捷键

1. **按 Win + X**
2. **选择**：
   - "Windows PowerShell"
   - 或 "Windows PowerShell (管理员)"（需要管理员权限）
   - 或 "终端"

### 方法 4：使用运行对话框

1. **按 Win + R**
2. **输入** `powershell` 或 `cmd`
3. **按回车**

---

## 第二步：验证 Node.js 是否已安装

在 PowerShell 或命令提示符中运行：

```bash
node --version
npm --version
```

如果显示版本号（如 `v20.11.0`），说明已安装，可以继续下一步。

如果显示 "无法识别" 或 "不是内部或外部命令"，需要先安装 Node.js：

### 安装 Node.js

1. 访问：https://nodejs.org/zh-cn/
2. 下载 LTS 版本（推荐长期支持版本）
3. 运行安装程序，全部选择默认选项
4. 安装完成后，**重新打开** PowerShell 或命令提示符

---

## 第三步：安装项目依赖

在 PowerShell 中，确保你在项目目录 `F:\webtest`，然后运行：

```bash
npm install
```

这个命令会下载所有需要的依赖包，可能需要几分钟时间。

---

## 第四步：启动开发服务器

依赖安装完成后，运行：

```bash
npm run dev
```

看到类似这样的输出就说明成功了：

```
✓ Ready in 2.3s
○ Local:        http://localhost:3000
```

---

## 第五步：查看网站

打开浏览器，访问：**http://localhost:3000**

---

## 常见问题

### 问题 1：提示 "npm 不是内部或外部命令"
**解决**：需要先安装 Node.js（见第二步）

### 问题 2：路径中包含中文导致错误
**解决**：使用引号包裹路径，如：`cd "F:\webtest"`

### 问题 3：端口 3000 已被占用
**解决**：关闭占用 3000 端口的程序，或修改 `package.json` 中的启动脚本

### 问题 4：安装依赖时速度很慢
**解决**：可以使用国内镜像源：
```bash
npm config set registry https://registry.npmmirror.com
```

---

## 停止开发服务器

在 PowerShell 中按 `Ctrl + C` 可以停止开发服务器。
