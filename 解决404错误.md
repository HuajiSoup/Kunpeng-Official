# 解决 404 错误

## 检查清单

### 1. 确认开发服务器正在运行

在 cmd 窗口中，你应该看到类似这样的输出：
```
▲ Next.js 14.2.0
- Local:        http://localhost:3000
- Ready in X.Xs
```

**如果没有看到这个输出，说明服务器没有成功启动。**

### 2. 检查访问的 URL

确保你在浏览器中访问的是：
- ✅ `http://localhost:3000` 
- ❌ 不要访问 `http://localhost:3000/about` 或其他路径（这些页面还没创建）

### 3. 查看 cmd 窗口的错误信息

如果服务器没有启动，cmd 窗口会显示错误信息。请查看：
- 是否有编译错误？
- 是否有端口被占用的提示？
- 是否有模块找不到的错误？

### 4. 尝试重启服务器

在 cmd 窗口中：
1. 按 `Ctrl + C` 停止当前服务器
2. 运行 `npm run dev` 重新启动
3. 等待看到 "Ready" 消息
4. 然后访问 `http://localhost:3000`

## 常见问题和解决方案

### 问题 1: 端口 3000 被占用

如果看到 "Port 3000 is already in use"：

**解决方案：**
1. 关闭占用端口的程序
2. 或者使用其他端口：`npm run dev -- -p 3001`
3. 然后在浏览器访问 `http://localhost:3001`

### 问题 2: 编译错误导致服务器无法启动

如果 cmd 窗口显示编译错误：

**解决方案：**
1. 清理缓存：删除 `.next` 文件夹
2. 重新启动：`npm run dev`

### 问题 3: 模块找不到

如果看到 "Cannot find module" 错误：

**解决方案：**
```bash
# 重新安装依赖
rmdir /s /q node_modules
del package-lock.json
npm install
npm run dev
```

### 问题 4: 访问了错误的路径

确保访问首页：`http://localhost:3000`

其他页面（如 `/about`、`/testing`）还没有创建，所以会显示 404。
